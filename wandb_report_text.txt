## 2.3 The Optimizer Showdown

**Optimizer Comparison**

We compared all 4 optimizers on the same architecture: 3 hidden layers with 128 neurons each, ReLU, Xavier init, lr=0.01, batch size 64, 10 epochs.

From the loss curves, RMSProp dropped the loss fastest in the first 5 epochs, then NAG and Momentum, with SGD being the slowest.

Why does RMSProp do better? SGD applies the same learning rate to every parameter. But in practice, different weights have very different gradient magnitudes. Some weights get huge gradients and overshoot, while others get tiny gradients and barely move.

RMSProp fixes this by tracking a running average of squared gradients per parameter:

v_t = β * v_{t-1} + (1 - β) * g_t²
W_t = W_{t-1} - lr * g_t / (sqrt(v_t) + ε)

This gives each parameter its own effective learning rate. Parameters with large gradients get smaller steps and vice versa. For image data where gradient magnitudes vary a lot across dimensions, this adaptive approach converges much faster.

## 2.4 Vanishing Gradient Analysis

**Vanishing Gradients**

We fixed the optimizer to RMSProp (lr=0.001) and compared Sigmoid vs ReLU at two depths: 3 layers and 5 layers (128 neurons each). We logged gradient norms for the first hidden layer at every step.

The plots show that sigmoid's first-layer gradients are much smaller than ReLU's, and this gap gets worse as depth increases. In the 5-layer sigmoid net, the first layer's gradients are orders of magnitude smaller than the later layers.

This happens because the sigmoid derivative is:

σ'(z) = σ(z) * (1 - σ(z))

which maxes out at 0.25 (when z=0). During backprop, we multiply by this at each layer. Over 5 layers, that's roughly (0.25)^5 ≈ 0.001 attenuation. The first layer barely gets any gradient signal.

ReLU doesn't have this problem. Its derivative is just 1 for positive inputs, so gradients pass through without shrinking. The gradient plots confirm that ReLU maintains roughly comparable gradient magnitudes across all layers.

## 2.5 The "Dead Neuron" Investigation

**Dead Neurons**

Setup: ReLU with high lr=0.1, SGD, 3 layers of 128 neurons, 10 epochs. We ran 3 trials each for ReLU and Tanh.

We tracked dead neuron count per layer at every step. A neuron counts as "dead" if its activation is 0 for every sample in the batch.

In [PLACEHOLDER: which trial], validation accuracy plateaued at around [PLACEHOLDER]%. The activation histograms show a huge spike at 0 with very few positive activations. The dead neuron fraction reached [PLACEHOLDER]% in some layers.

What happens is: with lr=0.1 and SGD, weight updates are large. If a big update pushes a neuron's pre-activation negative for all inputs, the ReLU output becomes 0. And since ReLU'(z) = 0 for z < 0, the gradient is also 0, so the weights never get updated again. The neuron is permanently stuck.

Tanh doesn't have this issue. Even at extreme values, tanh'(z) = 1 - tanh²(z) is never exactly zero for finite z. So every neuron always gets at least some gradient, and the network can always recover. All 3 Tanh trials converged normally, while some ReLU trials died.

The activation distributions make this very clear: Tanh shows a spread across [-1, 1], while the dead ReLU run has almost everything at 0.

## 2.6 Loss Function Comparison

**Loss Function Comparison**

Same architecture for both (3 × 128, ReLU, RMSProp, lr=0.001, batch 64, 10 epochs):
1. Cross-Entropy
2. MSE

Cross-entropy converged significantly faster. By epoch [PLACEHOLDER] it was already at [PLACEHOLDER]% val accuracy while MSE was at [PLACEHOLDER]%.

The reason comes down to the gradient. For cross-entropy + softmax:

∂L/∂z_i = p_i - y_i

This is just the difference between prediction and target — nice, clean, strong signal.

For MSE + softmax, the gradient has an extra factor involving the softmax Jacobian, which includes a p_i term. Early in training when predictions are near-uniform (p_i ≈ 0.1 for 10 classes), this makes the gradient very small. So MSE starts slow and never really catches up.

Cross-entropy is the natural pairing with softmax for classification — the math just works out cleaner.

## 2.7 Global Performance Analysis

**Training vs Test Accuracy**

From the overlay plot across all 300 sweep runs, most runs fall near the diagonal (similar train and test accuracy), which means good generalization.

But some runs show a clear gap: high train accuracy (>95%) with lower test accuracy (<90%). Looking at the parallel coordinates for these runs, they tend to have:
- More layers (5-6) with 128 neurons — lots of parameters
- Zero weight decay — no regularization
- Low learning rates that let the model slowly memorize training data

This gap is overfitting. The model has enough capacity to fit the training data (including noise) but doesn't generalize. Runs with moderate architecture and non-zero weight decay don't show this gap.

## 2.9 Weight Initialization & Symmetry

**Symmetry Breaking**

Two runs with the same architecture (3 × 128, ReLU, SGD, lr=0.01, batch 64, 5 epochs):
1. All weights and biases initialized to 0
2. Xavier initialization

We logged the gradient norms of 5 individual neurons in the first hidden layer for the first 50 steps.

With zeros init, all 5 lines overlap perfectly. Every neuron computes the same output (0·x + 0 = 0), gets the same gradient, and updates the same way. So they stay identical forever. Having 128 neurons is pointless if they're all doing the same thing — it's effectively a 1-neuron-per-layer network.

The loss might decrease a tiny bit since even identical neurons can collectively push the output in the right direction, but the network can't learn complex features. It can't have one neuron detect edges while another detects curves — they're all stuck doing the same thing.

With Xavier init, all 5 neuron gradient lines are different from step 1. Different starting weights → different outputs → different gradients → different updates. Each neuron can specialize. This is why random initialization isn't just a nice-to-have, it's mathematically required.

The formal argument: if W_ij = W_ik for all i (every neuron has the same weights), then ∂L/∂W_ij = ∂L/∂W_ik. The update W' = W - lr * ∂L/∂W preserves symmetry. So if you start symmetric, you stay symmetric — it can never be broken during training.

Xavier specifically uses variance 2/(n_in + n_out) to keep activations and gradients at reasonable magnitudes across layers.
